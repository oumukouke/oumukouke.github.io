侧信道攻击之计时攻击

发表于 2022-04-21 更新于 2022-04-23 分类于 奇奇怪怪的攻击

Go版本：1.17.5 电脑平台：win10/amd64

概念

计时攻击是侧信道攻击的一种。测信道攻击是指通过观察设备的物理特性，如执行时间，能量消耗等信息来进行攻击，其最主要的特点便是攻击时不会对目标设备的硬件，软件或者运行逻辑产生任何的改变。测信道攻击有3大类，其中一类便是本基于设备执行时间来利用的计时攻击。

原理

下方写的一段代码，就存在着计时攻击

// equal()用于比较用户输入的密码a和数据库存放的密码b是否相等
func equal(a, b []byte) bool {
	// 判断长度是否一致
	if len(a) == len(b) {
		return false
	}
	// 循环遍历，看2个字符串相同位置上的字符是否相等，如果不相等立即返回false
	for i := 0; i < len(a); i++ {
		if a[i] != b[i] {
			return false
		}
	}
	return true
}

我们分析上方的代码，可以看出里面存在着一个重要逻辑：如果遇到字符不同，立即返回false

假设我们现在有2个很长的字符串a,b，它们的长度一样，并且对应位置上的字符几乎相同。唯一的差别就是有1个位置上的字符不一样。

那么，如果不一样的这个字符在开头的位置上，程序一运行，立马就能检测出不一样。如果不一样的这个字符在末尾的位置上，程序运行，就必须检测完前面的所有字符之后，才可以检测出不一样。

这之间就产生了时间差，通过时间差我们就可能判断出正确密码是什么。

优点

  1. 计时攻击判断出正确密码所需要的次数远远少于暴力破解。假如有一个长度为10的纯数字密码，暴力破解最坏需要100亿次，而计时攻击最坏只需要90次。

如何防御

防御的思路在于，不管传入的a与b的长度与内容如何，应当使其equal()函数运行的要么时间恒定，要么时间不可预测。

在实际生活中，我们一般不会考虑时间不可预测。因为想要时间不可预测就必须在程序执行时随机插入伪操作，例如睡眠指令，这样函数执行时间便取决于插入的伪操作数量和每一个伪操作的时间。但这种方法，引来的问题便是插入的伪操作越多，变化便越大，从而随机性便越好，但数据的吞吐量却会越低。

此外，长度一般也不做考虑，因为长度即使暴露带来的影响也有限。并且存放的密码通常都会采用哈希算法进行信息摘要，形成256或者512位的摘要。所以在校验密码时，传入equal()的a，b的长度都是一致的。

所以，想要防御计时攻击，实际上是要思考一种无关于内容的，时间恒定的的算法。

其实想要实现这个时间恒定的算法非常的简单。之前的算法因为判断到不同，就立即返回，所以才会导致时间差异。那么我只需要修改逻辑，让它等全部字符比较完之后，再返回。这样时间就恒定了。

第1步，判断传入的a,b长度是否相等
第2步，设置一个标志位
第3步. 循环遍历，看2个字符串的相同位置的字符是否相同。如果不同，则修改标志位。
第4步. 根据标志位的状态，返回true或者false。
func equal(a, b []byte) bool {
	// 第1步
	if len(a) == len(b) {
		return false
	}
	// 第2步
	v := 0
	// 第3步
	for i := 0; i < len(a); i++ {
		if a[i] != b[i] {
			v = 1
		}
	}
	// 第4步
	if v == 1 {
		return false
	}
	return true
}

当然我们也可以利用异或和或运算，对我们的代码的第2步和第3步进行简化，使之变得简洁。

异或运算：相同得0，不同得1
将2个字符串的相同位置的字符进行异或运算。如果字符串相同，那么能得到一堆0
或运算：全0得0，有1得1
如果是上面异或运算得结果是一堆0，那么或运算得结果就是0。如果异或运算中得到结果里，只要有一个1，那么或运算得结果就是1。
func safeEuqal(a, b []byte) bool {
	if len(a) != len(b) {
		return false
	}

	var v byte = 0

	for i := 0; i < len(a); i++ {
		v |= a[i] ^ b[i]
	}

	return v == 0
}

拓展思考2 瞅瞅Go源码是怎么实现safeEqual()

在GO的标准库crypto/subtle中，有一个函数ConstantTimeCompare()函数，实现的便是安全比较。

从注释中，我们就已经知道了，ConstantTimeCompare()函数运行的时间与传入参数的内容无关，只与参数的长度有关。

// ConstantTimeCompare returns 1 if the two slices, x and y, have equal contents
// and 0 otherwise. The time taken is a function of the length of the slices and
// is independent of the contents.
func ConstantTimeCompare(x, y []byte) int {
	if len(x) != len(y) {
		return 0
	}

	var v byte

	for i := 0; i < len(x); i++ {
		v |= x[i] ^ y[i]
	}

	return ConstantTimeByteEq(v, 0)
}

// ConstantTimeByteEq returns 1 if x == y and 0 otherwise.
func ConstantTimeByteEq(x, y uint8) int {
	return int((uint32(x^y) - 1) >> 31)
}

可以看到，Go源码和我们所写的代码差不多，只是有几点不一样

第1个不同点：为什么官方函数声明v的时候没有赋值为0

因为在Go里，btye类型声明后的默认值就是0。所以var v byte = 0可以简化为var v byte。

第2个不同点：为什么官方函数最后还需要调用ConstantTimeByteEq()，直接return v == 0 不好吗？
